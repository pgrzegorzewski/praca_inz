Symmetric Cryptosystems

In these cryptography systems both sender and recipent have the same key. They are obliged to choose key before the transmission of a message.

Sender uses key for message encryption. Recipent uses the same key for decrypting messages. That is why we call this system symmetric, we have got one key which is used for both encryption and decryption.
Cryptosytem which uses symmetric keys are for example Caesar cipher.


This is the biggest drawback of these systems, keys for both sides are exactly the same.
However these systems have got one crucial advantage over the systems with asymmetric keys which is lower computational complexity of encrypting and decrypting procedures (when we compare two cryptograms at the same level of safety).
