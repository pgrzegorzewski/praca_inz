Cryptosystem with symmetric keys

In this cryptography system both sender and recipent have the same key. They are obliged to choose key before the transmission of the message.

Sender uses key for encrypting message. Recipent uses the same key for decrypting messages. That is why we call this system symmetric, we have got one key which is used for both encryption and decryption.
Cryptosytems which are using symmetic keys are for example RSA cipher


This the biggest drawback of the system, key for both sides is exactly th same.
However this system has got one crucial advantage over the system with asymetric keys which is lower computational complexity of encrypting and decrypting procedures (when we are comparing two cryptograms at the same level of safety).
