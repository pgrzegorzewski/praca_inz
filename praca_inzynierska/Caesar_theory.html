Ceasar Cipher

Historical brief

Caesar Cipher is the first cipher wich appears in human history. We can find the same cryptology system named "shift cipher" or "Caesar code" The name of cipher comes from the first user of that ciphr - Julius Ceasar. That cipher was used to encrypt mainly military orders. There was used encryption with right shift of 3. In historical documents we can find that Caesar used constant shit "three". For that time Caesar's cipher was a huge milestone in cryptology. Nobody expected that paper covered with as may have seen at first look random letters can hide a secret message.

Despite the fact that Caesar cipher does not provide any safety it is still used nowadays with right shift of 13 (ROT13). We can find this type of cipher in few online communicators and forums.

Cipher description

Caesar cipher belongs to an affine, symmetric, subtitution family of ciphers. In this cipher every single character is replaced with character from shifted alphabet. Key in this cipher is represented by a number which is the shift of alphabet.
Processes of encryption and decryption look almost identically.

For changing text into cryptogram we are using following formula:
En(x) = (x + n) mod 26
Where:
En - encrypted character
x - following characters
n- shift or key
We are using mod 26 baceause English alphabet consists of 26 characters. We can expand alphabet by special characters like white characters or numbers.

For decrypting text we are using formula:
Dn(x) = (x - n) mod 26
Where:
Dn - decrypted text
x - following character
n - shift

Cryptoanalysis methods


Unfortunately Caesar cipher was useful only in for small period before Christ when nobody had heard about ciphers. Today everyone can break the cipher in few minutes with a pencil and piece of paper.
Caesar cipher does not provide any safety. However other ciphers which are using this simple cryptology method can be even unbreakable (of course some conditions must be fullfilled).
To break the cipher we can use:
  • Brute force method - checking all existing keys one by one
  • Finding element of decrypted text
  • Statistics - with knowledge which letters are the most common for language we can suppose which encrypted letters may be in plain text.
